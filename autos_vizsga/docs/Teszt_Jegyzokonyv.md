BevezetésJelen jegyzőkönyv a CarCore webáruház backend API-jának automatizált integrációs tesztelését dokumentálja. A tesztelés elsődleges célja az API végpontok stabilitásának, a megfelelő válaszkódoknak és az adatszerkezetek integritásának ellenőrzése volt az élesítést megelőzően.2. Tesztkörnyezet és EszközökOperációs rendszer: WindowsFuttatókörnyezet: Node.js (v18+)Tesztelési keretrendszerek: Jest, SupertestAdatforrás: Lokális JSON állományok (fejlesztői környezet)3. Tesztesetek (Test Cases)TC01: Egészségellenőrzés (Health Check)Cél: A backend API alapvető elérhetőségének és válaszképességének ellenőrzése.Előfeltételek: A Node.js szerver futása.Lépések: 1. GET típusú HTTP kérés indítása az /api végpontra.Elvárt kimenet: - 200 OK HTTP státuszkód.A válasz tartalmazza a megfelelő szerverstátuszt (pl. szöveges üzenet vagy JSON objektum).Tényleges eredmény: ✅ SIKERES (PASSED)TC02: Termékkatalógus ListázásaCél: A terméklista sikeres lekérdezésének és az adatszerkezet validitásának ellenőrzése.Előfeltételek: A szerver futása és a data/products.json fájl elérhetősége.Lépések: 1. GET típusú HTTP kérés indítása az /api/products végpontra.Elvárt kimenet: - 200 OK HTTP státuszkód.A válasz formátuma egy JSON tömb.A tömb elemei tartalmazzák a kötelező attribútumokat (pl. id és name).Tényleges eredmény: ✅ SIKERES (PASSED)TC03: Hibakezelés Validációja (404 Not Found)Cél: A rendszer válaszának ellenőrzése érvénytelen vagy nem létező útvonal meghívása esetén.Előfeltételek: A szerver futása.Lépések: 1. GET típusú HTTP kérés indítása a nem létező /api/nemletezo végpontra.Elvárt kimenet: - 404 Not Found HTTP státuszkód visszaadása.Tényleges eredmény: ✅ SIKERES (PASSED)4. Összegzés és EredményekAzonosítóTeszteset megnevezéseStátuszMegjegyzésTC01API elérhetőség✅ PASSEDA végpont megfelelően válaszol.TC02Terméklista lekérése✅ PASSEDHelyes JSON adatszerkezet.TC03Érvénytelen útvonal✅ PASSEDA hibakezelés az elvárásoknak megfelelő.(Ide illeszthető egy képernyőfotó a terminálról, amely a zöld pipákat és a sikeres Jest lefutást ábrázolja)5. KonklúzióAz elvégzett automatizált integrációs tesztek alapján a CarCore API alapvető funkciói stabilan működnek. A rendszer a valid kéréseket megfelelően kiszolgálja, az érvényteleneket pedig biztonságosan lekezeli. Az API a jelenlegi formájában alkalmas a vizsgabemutatóra és a frontend klienssel való megbízható kommunikációra.6.
